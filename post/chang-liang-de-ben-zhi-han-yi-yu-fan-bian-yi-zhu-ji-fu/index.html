常量的本质含义与反编译助记符

2019-05-20

案例

public class MyTest2 {
    public static void main(String[] args) {
        System.out.println(MyParent2.src);
    }
}

class MyParent2{
    public static final String src = "hello world";

    static {
        System.out.println("MyParent2 static block");
    }
}

运行结果

hello world

运行上述代码，猜测下MyParent2会不会被加载以及被初始化。哈哈，我既然这么问了，答案肯定就是否定的，我们一起分析下原因。

首先，我们还是先看一下JVM加载了哪些类，如下图所示： 从图中可知，JVM只加载了com.ryan.jvm.classloader.MyTest2,因为这是启动类（主动使用），所以被虚拟机加载了，但是MyParent2没有被加载，是不是意味着我们并不需要MyParent2.class文件呢，带着大胆的想法，于是我把target目录下生成的MyParent2.class文件删除，重新运行一次代码，运行结果依旧是hello world。 好神奇，于是我决定反汇编一探究竟。 很明显的是，ldc直接将常量hello world压栈，并没有任何MyParent2的影子。 为了实验对比，我们现在可以把public static final String src = "hello world"改为public static String src = "hello world"，重新编译之后我们再来反汇编，得到下图： 不难发现MyParent2.src的影子 于是，我们不难得出一个结论：常量在编译阶段会存入到调用这个常量的方法所在类的常量池中，本质上，调用类并没有直接引用到定义常量的类，因此并不会触发定义常量的类的初始化（这里指的是将常量存放到了MyTest2的常量池中，之后MyTest2与MyParent2就没有任何关系了甚至，我们可以将MyParent2.class文件删除）

彩蛋

分享下在JVM中int类型数值采用何种指令入栈的，根据int值范围JVM入栈字节码指令就分为4类，下面分别介绍下这四类指令。 当int取值-1~5采用iconst指令iconst_m1~iconst_5，取值-128~127采用bipush指令，取值-32768~32767采用sipush指令，取值-2147483648~2147483647采用ldc指令。
